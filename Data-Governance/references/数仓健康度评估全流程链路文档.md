## 二、各专项评估链路详情

### 2.1 模型规范检查 (skill_1)

#### 2.1.1 输入

| 输入类型 | 说明 | 格式要求 |
|---------|------|---------|
| 表清单文件 | 待检查的表信息 | 放在 `input/` 目录下 |
| 建表DDL | 表结构描述 | 文本文件，每行一个表信息 |
| 表结构描述 | 包含表名、字段、类型、注释 | 支持多种格式 |

#### 2.1.2 执行步骤

```
步骤1: 读取输入文件
    │
    ▼
步骤2: 层级识别（根据表名前缀）
    │   ods_ → ODS层
    │   dwd_ → DWD层
    │   dws_ → DWS层
    │   dim_ → DIM层
    │   ads_ → ADS层
    │
    ▼
步骤3: 应用检查规则
    │   ├─ 表命名检查
    │   ├─ 字段规范检查
    │   ├─ 分区规范检查
    │   ├─ 存储规范检查
    │   └─ 文件名检查（仅ODS层）
    │
    ▼
步骤4: 计算扣分和得分
    │   总扣分 = 表命名扣分 + 字段扣分 + 分区扣分 + 存储扣分 + 文件名扣分
    │   最终得分 = 100 - 总扣分
    │
    ▼
步骤5: 生成CSV报告
```

#### 2.1.3 各层检查规则详情

| 层级 | 表命名格式 | 检查项 | 分值 |
|-----|-----------|--------|-----|
| ODS | `ods_{源系统}_{表名}_{增量标识}` | 表命名/字段/分区/存储/文件名 | 50分 |
| DWD | `dwd_{业务域}_{表名}_{增量标识}` | 表命名/字段/分区/存储 | 45分 |
| DWS | `dws_{业务域}_{汇总主题}_{增量标识}` | 表命名/字段/分区/存储 | 45分 |
| DIM | `dim_{维度名}` | 表命名/字段/存储 | 60分 |
| ADS | `ads_{应用主题}_{表名}` | 表命名/字段/分区/存储 | 45分 |

#### 2.1.4 产出

| 产出类型 | 说明 | 位置 |
|---------|------|------|
| CSV报告 | 所有表的检查结果汇总 | `output/dw_healthy_check.csv` |

#### 2.1.5 CSV输出字段

| 字段 | 说明 |
|-----|------|
| 表名 | 检查的表名称 |
| 层级 | ODS/DWD/DWS/DIM/ADS/未知 |
| 表命名检查扣分 | 表命名检查项扣分值 |
| 字段规范检查扣分 | 字段规范检查项扣分值 |
| 分区规范检查扣分 | 分区规范检查项扣分值 |
| 存储规范检查扣分 | 存储规范检查项扣分值 |
| 文件名检查扣分 | 文件名检查项扣分值（仅ODS层） |
| 总扣分 | 各项扣分合计 |
| 最终得分 | 100-总扣分 |
| 问题描述 | 该表所有问题的详细描述 |
| 分区建议 | 分区设计建议（仅DIM层） |

### 2.2 代码检查 (skill_2)

#### 2.2.1 前置依赖

| 依赖项 | 说明 |
|-------|------|
| db_config.json | 数据库连接配置（source_db + target_db） |
| data_syncer.py | 数据同步工具 |
| rdos_batch_task表 | 存储待检查的SQL代码 |
| code_rules.md | 代码检查规则集 |

#### 2.2.2 输入

| 输入类型 | 说明 | 来源 |
|---------|------|------|
| SQL代码 | 待检查的SQL语句 | 从rdos_batch_task表读取 |
| code_id | 代码唯一标识 | rdos_batch_task.id |
| code_layer | 代码所属分层 | 根据表名/任务类型识别 |

#### 2.2.3 执行步骤

```
步骤1: 从数据库获取待检查代码
    │   SELECT id, name, sql_text, task_type FROM rdos_batch_task
    │   WHERE is_deleted = 0
    │
    ▼
步骤2: 识别代码所属分层
    │   ├─ 根据表名识别：ods_/dwd_/dws_/dim_/ads_
    │   └─ 根据task_type识别
    │
    ▼
步骤3: 执行5大类规则检查
    │   │
    │   ├─ 1. 安全合规检查(6条)
    │   │     • 敏感数据检查
    │   │     • SQL注入检查
    │   │     • 权限检查
    │   │
    │   ├─ 2. 性能优化检查(6条)
    │   │     • 全表扫描检查
    │   │     • 笛卡尔积检查
    │   │     • 缺少分区剪裁
    │   │
    │   ├─ 3. 分层原则检查(4条)
    │   │     • ODS层检查
    │   │     • DWD层检查
    │   │     • DWS/ADS层检查
    │   │
    │   ├─ 4. 代码规范检查(22条)
    │   │     • 命名规范
    │   │     • 注释规范
    │   │     • 格式规范
    │   │
    │   └─ 5. SQL反模式检查(6条)
    │          • SELECT * 检查
    │          • 隐式类型转换
    │          • 负向查询
    │
    ▼
步骤4: 生成检查结果
    │   写入 code_check_result 表
    │
    ▼
步骤5: 计算评分
        代码得分 = 100 - (BLOCK×25) - (WARN×10) - (INFO×3)
```

#### 2.2.4 规则分类详情

| 规则类型 | 规则数量 | 优先级 | 扣分权重 |
|---------|---------|--------|---------|
| 安全合规检查 | 6条 | P0 | BLOCK |
| 性能优化检查 | 6条 | P1 | BLOCK/WARN |
| 分层原则检查 | 4条 | P2 | WARN |
| 代码规范检查 | 22条 | P3 | WARN/INFO |
| SQL反模式检查 | 6条 | P4 | INFO |

#### 2.2.5 产出

| 产出类型 | 说明 | 位置 |
|---------|------|------|
| 检查结果表 | 每条SQL的检查结果 | code_check_result表 |
| 分层评分 | 按ODS/DWD/DWS/DIM/ADS分层统计 | SQL查询结果 |

#### 2.2.6 code_check_result表结构

| 字段 | 说明 |
|-----|------|
| code_id | 代码ID |
| code_name | 代码名称 |
| code_layer | 所属分层 |
| rule_id | 规则ID |
| rule_name | 规则名称 |
| rule_severity | 严重程度(BLOCK/WARN/INFO) |
| check_status | 检查结果(PASS/FAIL) |
| issue_location | 问题位置 |
| issue_desc | 问题描述 |
| suggestion | 整改建议 |

### 2.3 数据质量检查 (skill_3)

#### 2.3.1 前置依赖

| 依赖项 | 说明 |
|-------|------|
| db_config.json | 数据库连接配置 |
| db_connector.py | 数据库连接工具 |
| meta_table_column表 | 存储表字段元数据 |
| dq_check_result表 | 存储校验结果 |
| rule_v2.md | 质量校验规则集 |

#### 2.3.2 输入

| 输入类型 | 说明 | 来源 |
|---------|------|------|
| 数据库连接 | 数仓数据库 | 从db_config.json读取 |
| 表清单 | 待检查的表 | 自动扫描以ods/dwd/dws/dim/ads开头的表 |
| 字段信息 | 表字段元数据 | 从meta_table_column表读取 |
| 校验规则 | 质量规则 | 从rule_v2.md读取 |

#### 2.3.3 执行步骤

```
步骤0: 初始化环境
    │   ├─ 创建meta_table_column表
    │   ├─ 创建dq_check_result表
    │   └─ 同步元数据
    │
    ▼
步骤1: 同步元数据
    │   ├─ 连接数仓数据库
    │   ├─ 扫描以ods/dwd/dws/dim/ads开头的表
    │   ├─ 提取每个表的字段信息(名称、类型、注释)
    │   └─ 写入meta_table_column表
    │
    ▼
步骤2: 识别字段标签
    │   ├─ DIMENSION: name/code/type/status/flag/org/dept/region/date/time
    │   ├─ MEASURE: mny/amount/price/cost/qty/count/rate/percent
    │   └─ ATTRIBUTE: 其他普通字段
    │
    ▼
步骤3: 为每个表生成校验SQL
    │   │
    │   ├─ 通用规则(COM_001~COM_007)
    │   │     • 分区连续无缺失
    │   │     • 数据量波动合理
    │   │     • 关键字段非空率
    │   │     • 时间字段无异常
    │   │     • 脏数据/乱码检查
    │   │     • 金额非负检查
    │   │     • 枚举值合法性
    │   │
    │   └─ 分层专属规则
    │          • ODS专属规则
    │          • DWD专属规则
    │          • DWS专属规则
    │          • DIM专属规则
    │          • ADS专属规则
    │
    ▼
步骤4: 执行校验SQL
    │   ├─ 连接数据库
    │   ├─ 逐条执行校验SQL
    │   ├─ 记录执行结果(成功/失败/超时)
    │   └─ 写入dq_check_result表
    │
    ▼
步骤5: 生成评估报告
        ├─ 计算单表得分
        ├─ 计算分层得分
        └─ 计算整体数仓得分
```

#### 2.3.4 质量检查维度

| 维度 | 说明 | 检查内容 |
|-----|------|---------|
| 完整性 | 数据缺失程度 | 空值率、缺失率 |
| 准确性 | 数据准确程度 | 数值范围、枚举值 |
| 一致性 | 跨表数据一致性 | 主外键关系、关联字段 |
| 及时性 | 数据时效性 | 数据延迟天数 |
| 唯一性 | 主键唯一性 | 主键重复检查 |

#### 2.3.5 评分公式

**单表评分：**
```
单表得分 = 100 - Σ(规则扣分)
规则扣分 = 异常率 × 权重(100/25/10)
```

**分层评分：**
```
分层得分 = Σ(各表得分 × 表记录数) / Σ(表记录数)
```

**整体评分：**
```
整体得分 = Σ(各层得分 × 权重)
权重: ODS 20%, DWD 25%, DWS 20%, DIM 15%, ADS 20%
```

#### 2.3.6 产出

| 产出类型 | 说明 | 位置 |
|---------|------|------|
| 校验结果表 | 每张表的校验结果 | dq_check_result表 |
| 评估报告 | 健康度评估结果 | SQL查询输出 |

#### 2.3.7 dq_check_result表结构

| 字段 | 说明 |
|-----|------|
| table_name | 被校验的表名称 |
| table_layer | 表所属分层 |
| partition_value | 分区值 |
| total_rows | 表总记录数 |
| rule_id | 规则ID |
| rule_name | 评估规则名称 |
| rule_severity | 严重程度 |
| check_field | 校验的字段名称 |
| check_sql | 执行的评估SQL |
| execute_status | 执行状态 |
| check_status | 校验结果(PASS/FAIL) |
| anomaly_count | 异常记录数 |
| anomaly_rate | 异常率 |

### 2.4 血缘健康检查 (skill_4)

#### 2.4.1 输入

| 输入类型 | 说明 | 格式 |
|---------|------|------|
| Excel文件 | 包含SQL的Excel文件 | .xlsx格式 |
| SQL列 | Excel中的SQL内容列 | 文本 |
| sheet名 | Excel的sheet名称 | 字符串 |

#### 2.4.2 执行步骤

```
步骤1: 从Excel提取SQL
    │   ├─ 读取Excel文件
    │   ├─ 解析SQL列内容
    │   ├─ 转换为SQL AST
    │   └─ 提取表级血缘信息
    │
    ▼
步骤2: 解析血缘关系
    │   ├─ 解析SQL AST
    │   ├─ 识别表与表之间的依赖关系
    │   ├─ 识别字段级血缘关系
    │   └─ 输出血缘关系详情
    │
    ▼
步骤3: 评估血缘质量
    │   │
    │   ├─ 覆盖率评估
    │   │     • 血缘解析覆盖率
    │   │     • 未解析表列表
    │   │
    │   ├─ 正确性评估
    │   │     • 血缘关系正确性
    │   │     • 错误血缘列表
    │   │
    │   ├─ 完整性评估
    │   │     • 血缘链路完整性
    │   │     • 缺失链路列表
    │   │
    │   ├─ 稳定性评估
    │   │     • 血缘变动频率
    │   │     • 不稳定表列表
    │   │
    │   └─ 复杂度评估
    │          • 血缘链路复杂度
    │          • 复杂表列表
    │
    ▼
步骤4: 生成分析报告
        ├─ 表级血缘分析报告.md
        └─ 字段级血缘分析报告.md
```

#### 2.4.3 工具说明

| 工具 | 功能 | 输入 | 输出 |
|-----|------|------|------|
| 从Excel提取SQL | 解析Excel中的SQL | Excel文件路径 | 表级血缘关系详情.xlsx |
| 从SQL_AST解析血缘 | 解析SQL血缘关系 | Excel/血缘详情 | 血缘解析结果.xlsx |
| 生成血缘分析报告 | 输出分析报告 | 解析结果 | .md报告文件 |
| 影响范围分析 | 分析修改影响范围 | 表名/字段名 | 影响范围报告.md |

#### 2.4.4 评估指标详情

| 指标 | 说明 | 评分标准 |
|-----|------|---------|
| 覆盖率 | 血缘关系解析覆盖率 | 覆盖率越高越好 |
| 正确性 | 血缘关系正确性评估 | 错误率越低越好 |
| 完整性 | 血缘链路完整性 | 完整率越高越好 |
| 稳定性 | 血缘变动频率 | 稳定率越高越好 |
| 复杂度 | 血缘链路复杂度 | 复杂度适中最好 |

#### 2.4.5 产出

| 产出类型 | 说明 | 格式 |
|---------|------|------|
| 表级血缘关系详情 | 每张表的血缘信息 | Excel |
| 血缘解析结果 | 解析后的血缘数据 | Excel |
| 表级血缘分析报告 | 血缘现状分析 | .md |
| 字段级血缘分析报告 | 字段血缘详情 | .md |
| 影响范围分析报告 | 修改影响分析 | .md |

## 三、综合评估链路

### 3.1 整体执行流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    数仓健康度评估完整流程                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  【阶段一：准备工作】                                             │
│    1. 确认配置文件 db_config.json                                 │
│    2. 确认工具脚本可用 (data_syncer.py, db_connector.py)         │
│    3. 准备待检查数据                                             │
│       • skill_1: input/目录下的表清单                             │
│       • skill_2: rdos_batch_task表中的SQL                        │
│       • skill_3: 数仓数据库中的表                                 │
│       • skill_4: 包含SQL的Excel文件                              │
│                                                                  │
│  【阶段二：执行专项评估】                                         │
│                                                                  │
│    skill_1(模型规范) ──▶ 步骤1~5 ──▶ CSV报告                     │
│         输入: 表清单                                             │
│         输出: dw_healthy_check.csv                              │
│                                                                  │
│    skill_2(代码检查) ──▶ 步骤1~5 ──▶ 结果表                       │
│         输入: rdos_batch_task表SQL                               │
│         输出: code_check_result表                                │
│                                                                  │
│    skill_3(数据质量) ──▶ 步骤0~5 ──▶ 结果表                      │
│         输入: 数仓数据库                                         │
│         输出: dq_check_result表                                  │
│                                                                  │
│    skill_4(血缘健康) ──▶ 步骤1~4 ──▶ Excel报告                    │
│         输入: Excel文件                                          │
│         输出: 血缘分析报告.md                                     │
│                                                                  │
│  【阶段三：汇总分析】                                             │
│    1. 收集各专项评估结果                                         │
│    2. 计算综合健康度评分                                         │
│    3. 生成综合健康度报告                                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 各专项输入输出汇总

| 专项 | 输入 | 中间产出 | 最终输出 |
|-----|------|---------|---------|
| skill_1 模型规范 | input/目录下的表清单 | 检查过程数据 | dw_healthy_check.csv |
| skill_2 代码检查 | rdos_batch_task表SQL | code_check_result表 | 分层评分统计 |
| skill_3 数据质量 | 数仓数据库 | dq_check_result表 | 健康度评估报告 |
| skill_4 血缘健康 | Excel文件 | 血缘解析结果.xlsx | 血缘分析报告.md |

### 3.3 综合评分

| 专项 | 权重 | 评分等级 |
|-----|------|---------|
| 模型规范 | 25% | 优秀(90+)/良好(75-89)/一般(60-74)/较差(<60) |
| 代码质量 | 25% | 优秀(90+)/良好(75-89)/一般(60-74)/较差(<60) |
| 数据质量 | 30% | 优秀(90+)/良好(75-89)/一般(60-74)/较差(<60) |
| 血缘健康 | 20% | 优秀(90+)/良好(75-89)/一般(60-74)/较差(<60) |

**综合评分公式：**
```
综合得分 = 模型规范×25% + 代码质量×25% + 数据质量×30% + 血缘健康×20%
```

## 五、执行命令参考

```bash
# 执行完整健康度评估
python health_check.py --all

# 执行指定专项评估
python health_check.py --modules model,code,quality,lineage

# 单独执行各专项
python health_check.py --modules model      # 模型规范检查
python health_check.py --modules code       # 代码检查
python health_check.py --modules quality    # 数据质量检查
python health_check.py --modules lineage    # 血缘健康检查
```
